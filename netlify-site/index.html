PET VILLAGE · PR #46

Camera zoom: scroll, pinch, and a right-side slider

Zoom into the village and minigames — closer by default, especially in games — without losing readable HUD chrome.

TL;DR

  • Web: mouse wheel / trackpad scrolls to zoom.
  • Mobile: two-finger pinch zooms (joystick and walk stay separate).
  • Slider: vertical control on the right edge (+ / −).
  • Hub opens at 1.25×; Skip Rope / Paper Toss / Fishing at 1.55×.
  • Range clamped 0.9×–2.0×. App version → 1.1.0.

Defaults

Minimum 0.9×
Hub default 1.25×
Game default 1.55×
Maximum 2.0×

What changed

  • Shared zoom: src/systems/cameraZoom.ts wires wheel, pinch, and the slider.
  • Scenes: Town, Shore, House, Shop, Cafe, Skip Rope, Paper Toss, Fishing.
  • UI camera: HUD, menus, joystick, Back, and the slider stay sharp while the world zooms.
  • Touch: up to 4 active pointers so joystick + pinch can coexist.
  • Gesture safety: pinch cancels walk / aim drag so zoom doesn’t start a throw or jump.
  • Memory: last hub zoom and last game zoom are remembered for the session.

How to try it

  1. Check out feat/camera-zoom and run npm run dev → /pet-village/.
  2. In Town, scroll the wheel or drag the right-edge slider — the plaza should zoom around you.
  3. On a phone, pinch with two fingers away from the joystick.
  4. Enter Skip Rope or Paper Toss — the view should start more zoomed in than town.
  5. Open a menu / check the HUD — labels and buttons should stay crisp, not scaled up with the world.
  6. Shell Menu should show v1.1.0.

Verification

Verified: TypeScript check (tsc --noEmit).
